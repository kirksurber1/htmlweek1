Website vs Web Application

Websites: for information purposes

Minimal data being passed other than maybe a capture form or link to another page

Web Apps

Transfer data between user and DB for productivity purposes

(e.g. Input customer data, schedule appointments, sales, shopping carts, etc. )

Grey Area

Websites can capture info and try to make sales much like Apps

This area can be debatable about functionality between the Apps and Sites on which is which.

Web Application

Multiple Request/Response API's

Data sent between user and backend database

(e.g. User adds item to cart on shopping on Amazon, collects payment/shipping info, backend stores info, generates a payment transaction, and a business will then fullfill the transaction. )

Data Transfer

User inputs data from computer

Backend storage or functionality is then implemented based on the programing. Whether storing a photo or sending a message, backed functionality is what allows the ability to do more than look at data
